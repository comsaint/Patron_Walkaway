# Phase 4：定版、回放與 Go / No-Go

## 為什麼要做這一階段

訓練與離線實驗達標不代表**上線安全**。Phase 4 在**凍結候選**後，用多時間窗與業務可讀指標驗證：**主指標是否仍達標、切片是否退化、告警量與誤報是否可承受**，並產出正式上線或延後的決策包。

## 如何調查

- **候選凍結**：鎖定資料窗、特徵集、模型 artefact、閾值與決策政策；凍結後只做回放與文檔更新，不再混入「順手優化」。
- **多窗回放**：至少多個具代表性的時間窗（含不同流量／週期特性）重跑 `precision@recall=1%` 與約定切片指標。
- **影響估算**：估算告警量、誤報量、對營運或業務 KPI 的敏感影響；必要時對照現行版本。
- **Go/No-Go 會議包**：彙總證據、已知風險、監控與回滾／降級設想，供利害關係人簽核。

與總計畫一致：主指標目標 **>= 60%**（`precision@recall=1%`），且須**跨窗穩定**，非單窗達標。

| `go_no_go_pack.md` | 最終建議、條件與後續行動 |

## 可依此做出的決策

- **Go（允許進入上線流程）**：主指標達標且**多窗一致**；**無不可接受之切片退化**（尤其高價值客群／高分段）；影響估算與監控方案可行。
- **No-Go（延後或退回）**：僅單窗達標、他窗明顯退化；或誤報／告警量超出可接受範圍；或風險無法被監控／回滾覆蓋。
- **條件式 Go**：在補齊特定監控、流量規模上限或 shadow 期間等**前置條件**後再上線（須寫入 `go_no_go_pack.md`）。
